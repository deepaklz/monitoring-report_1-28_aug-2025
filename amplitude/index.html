üîç Key Observations

Critical Issues Identified

  • Early abandonment: Sessions often end on input fields like CustomerName or SpectroReadings
  • Module-hopping is common: 82% of MTC sessions involve navigation to other modules
  • Form resets cause frustration: Users lose progress when navigating away to look up data
  • Data lookup limitations: Heat/test codes aren't searchable within the form
  • Low completion rate: Only 42% of certificates started are actually completed
  • Microanalysis bottleneck: Takes significantly longer (7+ minutes median) than other steps
  • No progress persistence: System wipes progress instead of saving drafts

üí° Recommendations

  • Integrate data lookup into MTC form ‚Äì Provide searchable drop-downs or modals that allow users to fetch spectro-chemical, hardness or tensile readings without leaving the form
  • Auto-save progress ‚Äì Persist all field values in browser or server. When users return to MTC form, previous inputs should be restored
  • Drafts and resume capability ‚Äì Display list of incomplete certificates with "Resume" button on listing page
  • Context-preserving navigation ‚Äì Allow opening other modules in new tab/modal instead of replacing form. Show confirmation before discarding unsaved changes
  • Guidance and progress indicators ‚Äì Provide contextual help on finding codes and display progress bar showing completion status
  • Training on heat/test codes ‚Äì Create guides or contextual help to reduce uncertainty about which codes to use
